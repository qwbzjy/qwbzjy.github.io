一条 SQL 语句查询是怎么执行的

Mysql 逻辑架构：Server 层和存储引擎。

Server 层由连接器、分析器、优化器、执行器、查询缓存 (MySQL 8.0已经将查询缓存的功能删掉) 组成，所有的内置函数(日期、时间、数学、加密函数等)，所有的跨存储引擎的功能，如存储过程、触发器、视图等都在 Server 层完成。

连接器：负责跟客户端建立连接、获取权限、维持和管理连接。(管理连接，权限验证)

长连接：指连接成功后，如果客户端持续有请求，则一直使用同一个连接。

短连接：指每次执行完很少的几次查询就断开连接，下次查询再重新建立一个。

注意事项：

  • 在使用中尽量减少建立连接的动作，也就是尽量使用长连接。
  • 长连接的影响：有时候 Mysql 占用的内存会涨的特别快，这是因为 MySQL 在执行过程中临时使用的内存是管理在连接对象里面的，这些资源会在断开连接的时候才会释放。如果长连接积累下来，可能导致内存占用太大，被系统强行杀掉(OOM)，从现象看就是 MySQL 异常重启了。
  • 对长连接的影响解决方法
    • 定期断开长连接，使用一段时间或者程序中执行过一个占用内存的大查询后，断开连接，然后重连
    • MySQL 5.7 及以上版本，可以在每次执行一个比较大的操作后，通过 mysql_reset_connection 来重新初始化连接资源。

分析器：词法分析，语法分析。词法分析就是对 SQL 语句中的字符串进行分析，比如，select * from t where ID=1，如识别到 select 关键字就判断其是查询语句，分析器会把字符串 “t” 识别成表名 t。语法分析就是根据 MySQL 的语法规则进行判断 SQL 语句是否符和语法规则。

优化器：执行计划生成，索引选择。优化器是在表里面有多个索引的时候，决定使用哪个索引，或者在一个语句有多表关联的时候，决定各个表的连接顺序。

执行器：操作引擎，返回结果。开始执行的时候，要先判断一下奴婢对这个表 t 有没有执行查询的权限，如果没有就会返回没有权限的错误。如果有权限，就会打开表继续执行。打开表的时候，执行器就会根据表的引擎定义，去使用这个引擎提供的接口。如上面的 sql 语句例子，执行器的流程：

1、调用 InnoDB 引擎接口取这个表的第一行，判断 ID 值是否是 1，如果不是则跳过，如果是则将这行存在结果集中

2、调用引擎接口取下一行，重复相同的判断逻辑，直到取到这个表的最后一行

3、执行器讲上述遍历过程中所有满足条件的行组成的记录集作为结果集返回给客户端

存储引擎：存储数据，提供读写接口。其架构模式是插件式的，支持InnoDB、MyISAM、Memory等多个存储引擎。从 MySQL 5.5.5版本开始，InnoDB 成为了默认的存储引擎。不同的存储引擎共用一个 Server 层，也就是从连接器到执行器的部分。
