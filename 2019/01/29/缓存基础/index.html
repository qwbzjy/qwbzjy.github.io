缓存基础

发表于 2019-01-29 | 分类于 spring框架 | | 阅读次数:
字数统计: 1,034 | 阅读时长 ≈ 4

缓存由来

主要作用：减少系统开销，提高系统效率。

文件缓存：数据存储在磁盘上，由于内存空间有限，以及掉电后内存会清空，会将数据持久化到硬盘上，达到释放内存空间和数据备份的目的。

内存缓存：数据存储在静态内存区域，无需进行磁盘 I/O操作，读写速度较快。

缓存名词

命中率：返回正确结果数/请求缓存次数，它是衡量缓存有效性的重要指标。

最大元素(最大空间)：缓存中可以存放的最大元素的数量，一旦超过这个数量，就会触发缓存启动清空策略。

清空策略：FIFO、LFU、LRU

  • FIFO(first in first out)

    ​ 先进先出策略，当缓存空间不够时，最先进入缓存的数据会被优先清除掉，适用于数据时效性较高的场景。

  • LFU(less frequently used)

    ​ 最少使用策略，当缓存空间不够时，最先清除掉使用次数最少的数据，也就是比较数据的命中次数，适用于对高频数据有效性要求较高的场景

  • LRU(least recently used)

    ​ 最近最少使用策略，无论是否过期，根据元素最后一次被使用的时间戳清除最远使用时间戳的元素，也就是比较元素最近一次被get的时间，适用于保证热点数据的有效性场景。

缓存分类

本地缓存(local cache)和分布式缓存(remote cache)

本地缓存：

​ 指的是在应用中的缓存组件，其最大的优点是应用和cache是在同一个进程内部，请求缓存非常快速，没有过多的网络开销等，在单应用不需要集群支持或者集群情况下各节点无需互相通知的场景下使用本地缓存较合适；同时，它的缺点也是应为缓存跟应用程序耦合，多个应用程序无法直接的共享缓存，各应用或集群的各节点都需要维护自己的单独缓存，对内存是一种浪费

分布式缓存:

​ 指的是与应用分离的缓存组件或服务，其最大的优点是自身就是一个独立的应用，与本地应用隔离，多个应用可直接的共享缓存。

缓存问题

  • 缓存穿透

    ​ 查询一个数据库一定不存在的数据时，请求会直接从数据库中进行查询。为了避免有人利用这种方式恶意攻击系统，因此必须对这种问题进行防范。

    ​ 解决方案：

    1. bloom filter(布隆过滤器)：利用所有可能的查询条件生成一个 bitmap ，在进行数据库查询之前会使用这个 bitmap 进行过滤，从而减轻数据库的访问压力。
    2. 空值缓存：在第一次查询完不存在的数据后，将该 key 与对应的空值放入缓存中，设置较短的失效时间，这样可以应对短时间的大量的该 key 攻击；如果不是人为恶意攻击，也无需过久的存储。

  • 缓存雪崩

    ​ 在同一时刻，所有的缓存同时失效，那么此时，所有系统的请求都会去查询数据库，导致系统瘫痪

    ​ 解决方案：

    1. 线程互斥：只允许一个线程构建缓存，其他线程等待，确保每个时刻只有一个线程在执行请求，从而减轻数据库压力，但是会降低系统的 qps。
    2. 交错失效时间： 错开不同的失效时间

  • 缓存击穿

    ​ 热点缓存达到失效时间时，大量的请求到达数据库，造成数据库崩溃。

    ​ 解决方案：

    1. 设置二级缓存，对于热点数据进行二级缓存，并对于不同级别的缓存设定不同的失效时间，防止缓存击穿。[LRU算法]
