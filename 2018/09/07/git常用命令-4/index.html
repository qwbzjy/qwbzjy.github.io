​ 这段时间终于忙完了，又是弄牙齿，又是忙新项目，难得今天有空闲学习时间，整理了一下spring-aop的知识点，做了几个demo熟悉了一下aop的相关知识。在学习的过程中，再一次体会到，不做学习笔记等于白学，今天碰到问题了，不记录下来，过几天又要重新学，时间成本太高，于是果断记录到博客上。

​ 今天在IDEA中新建好了spring-aop的项目，想着上传到github上，结果在IDEA中无法push的问题，总是提示could not read from remote repository，谷歌了一下，是IDEA中git的设置问题，在Settings->Version Control->Git中，将SSH executable设置为Native即可 。这样一设置，问题果然解决了，但是背后的原因还是没搞明白，姑且先记录到博客中，日后再来解决。

IDEA创建git仓库的过程：

1、场景：我自己在idea中新建了一个spring-aop的项目，项目做完后，我想将它上传到github上，然后上谷歌和百度找教程，然后自己整理成如下几张图，首先按照图1创建一个项目的本地仓库，创建完成后，会看到图2中.git文件；然后登陆github网站，新建一个Spring-Aop的仓库，紧接着在spring-prc目录下进入git命令窗口，输入图3中的命令，其中qwbzjy是我的github账户名，Spring-Aop是仓库名；本地库与远程库建立关联后，我试着pull,因为远程库中多了一个readme.txt文件，然后就报图4的错误，接着我输入git pull origin master –allow-unrelated-histories,然后接可以了，最后将我的代码push到远程库中,git push origin master。

​ 后来我想直接通过IDEA进行push结果失败，然后又进行一番搜索，最后进行这段文字上面截图的设置后，就成功push了。

​ 学习的过程是一个循序渐进的过程，只有这样一点一点的积累，才能够厚积薄发。
