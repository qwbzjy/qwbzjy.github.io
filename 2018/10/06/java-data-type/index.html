java-data_type

发表于 2018-10-06 | 分类于 java基础 | | 阅读次数:
字数统计: 1,624 | 阅读时长 ≈ 6

java 是一种静态类型语言(在编译时，每一个变量和表达式类型都是已知的)，也是一种强类型语言。java 有两种数据类型，原始数据类型(primitive data)和对象数据类型(object data)。

原始数据类型是单个值，没有特殊功能，java 有8中原始数据类型。

boolean

​ boolean 类型占用一个 bit，ture or false，它不能与其他数据类型进行转换。

byte

​ byte 类型是 8 位有符号二进制补码整数，值范围：-128-127。

short

​ short 类型是 16 位有符号二进制补码整数，值范围：-32768-32767。

int

​ int 类型是 32 位有符号二进制补码整数，值范围：-2^31-2^31-1

long

​ long 类型是 64 位有符号二进制整数，值范围：-2^63-2^53-1

float

​ float 类型是 32 位浮点型数，如 9.8f

double

​ double 类型是 64 位浮点数，如9.8d

char

​ char 类型是16 位无符号字符，值范围：’\u0000’ (0)-‘\uffff’ (65535)

值类型

​ 一个具有值类型（value type）的数据存放在栈内的一个变量中。即是在栈中分配内存空间，直接存储所包含的值，其值就代表数据本身。值类型的数据具有较快的存取速度。

引用类型

​ 一个具有引用类型（reference type）的数据并不驻留在栈中，而是存储于堆中。即是在堆中分配内存空间，不直接存储所包含的值，而是指向所要存储的值，其值代表的是所指向的地址。当访问一个具有引用类型的数据时，需要到栈中检查变量的内容，该变量引用堆中的一个实际数据。引用类型的数据比值类型的数据具有更大的存储规模和较低的访问速度。

装箱和拆箱

​ 装箱（值类型到引用类型）。将一个值类型变量装箱成一个引用类型变量，首先会在托管堆上为新的引用类型变量分配内存空间，然后将值类型变量拷贝到托管堆上新分配的对象内存中，最后返回新分配的对象内存地址。

​ 拆箱（引用类型到值类型）。 获取指向对象中包含值类型部分的指针，然后由程序员手动将其对应的值拷贝给值类型变量。

值传递和引用传递

​ 基本数据类型赋值都属于值传递,传递的是值的拷贝,值传递后,实参传递给形参的值,形参发生改变而不影响实参。即传递后就互不相关。

​ 引用类型之间赋值属于引用传递。引用传递的是对象的引用地址，就是变量所对应的内存空间的地址。也就是它的本身。就是将实参的地址传递给形参，形参改变了，实参当然被改变了，因为他们指向相同的地址。传递前和传递后都指向同一个内存空间。

java 内存中的堆与栈

​ 当一个方法执行时，每个方法都会建立自己的内存栈，在这个方法内定义的变量将会逐个放入这块栈内存里，随着方法的执行结束，这个方法的内存栈也会自然销毁，因此所有在方法中定义的变量都是放在栈内存中。栈的优势是，存取速度比堆要快，仅次于寄存器，栈数据可以共享。但缺点是，存在栈中的数据大小与生存期必须是确定的，缺乏灵活性。

​ 当我们在程序中创建一个对象的时候，这个对象将保存到运行时数据区，以便反复利用（因为对象的创建成本比较大），这个运行时数据区就是堆内存，堆内存中的数据不会随方法的结束而销毁，即使方法结束后，这个对象依然不会被销毁，只有当一个对象没有任何引用变量引用它时，系统的垃圾回收机制才会在合适的时候回收它。

java 堆中对象地址打印

​ 使用System.out.println(obj.hashcode())输出的时对象的哈希码，而非内存地址。在Java中是不可能得到对象真正的内存地址的，因为Java中堆是由JVM管理的不能直接操作。

​ 只能说此时打印出的Hash码表示了该对象在JAVA虚拟机中的内存位置，Java虚拟机会根据该hash码最终在真正的的堆空间中给该对象分配一个地址。但是该地址 是不能通过java提供的api获取的。

java 栈中变量的地址

​ 最近学习 c 语言，想到变量地址的问题，突然想到 java 中也有这样的问题，谷歌一搜索，大部分答案是: java 中不让程序员进行内存管理的，JVM 有自己的内存管理机制，这是 jvm 所特有的，如果你实在想打印栈变量的内存地址，可以自己重新写一个虚拟机。而且通常 java 开发中不会用到变量的地址，即使你拿到了，也没用，java 的设计初衷就是减少程序员的工作量，不让开发人员去操作指针，而是由 jvm 自己去管理内存。
